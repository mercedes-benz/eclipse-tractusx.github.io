EDC API Examples

API Spec​

The API spec of the EDC is constantly evolving. The full API documentation for each release can be viewed on management-api. The following are some example API calls for common use cases. They assume the default parameters from the previous local setup.

Create an Asset​

All objects in EDC are created by posting their JSON-serialized representation to the respective API input. Since most EDC objects are rather openly defined, most of the properties provided depend on the need of the individual user. Assets are no exception here.

URL

POST http://localhost:8080/api/v1/assets/

Body

{
  "asset": {
    "id": "asset1",
    "properties": {
      "exampleProperty": "exampleValue"
    }
  },
  "dataAddress": {
    "properties": {
      "URL": "www.path.to/the_asset"
    }
  }
}

Request an Asset Catalog​

To inspect the assets available to an EDC connector, we request its catalog.

URL

POST http://localhost:8080/api/v1/catalog/request

Body

{
  "providerUrl": "www.example.provider",
  "querySpec": {
    "filter": "AvailableWithPolicyXYZ",
    "limit": 0,
    "offset": 0,
    "sortField": "id",
    "sortOrder": "ASC"
  }
}
